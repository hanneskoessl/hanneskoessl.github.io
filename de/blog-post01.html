Big O-Notation

Wird verwendet, um Algorithmen zu vergleichen. Verglichen wird die Zeit- und Speicherkomplexität, also die Anzahl an benötigten Rechenoperationen und den benötigten Speicherplatz.

Python-Beispiele für unterschiedliche Zeitkomplexität:

O(n) 

            def print_items(n):
    for i in range(n):
        print(i)

print_items(10)

Konstanten fallen weg
O(2n) → O(n) 

            def print_items(n):
    for i in range(n):
        print(i)

    for i in range(n):
        print(i)

print_items(10)

O(n2) 

            def print_items(n):
    for i in range(n):
        for j in range(n):
            print(i, j)

print_items(10)

Nicht-dominante Terme weglassen
O(n2+n) → O(n2) 

            def print_items(n):
    for i in range(n):
        for j in range(n):
            print(i, j)

    for i in range(n):
        print(i)

print_items(10)

O(1) 

            def add_items(n):
    return n + n

print(add_items(10))

O(log n)
Ein Beispiel dafür wäre binäre Suche. Bei jedem Schritt wird der Suchbereich halbiert.

Eingabe von unterschiedlichen Termen
O(a+b) 

            def print_items(a, b):
    for i in range(a):
        print(i)
    
    for j in range(b):
        print(j)

print_items(3, 5)

Listen
my_list.append(15), my_list.pop() haben eine Zeitkomplexität von O(1). my_list.insert(5, 14) hat eine Zeitkomplexität von O(n).
